Somfy

Web Dev.

Brief

The Somfy project is a virtual showroom designed to simulate the functionality of a remote that controls automated curtains and blinds in real-time.

Being my first project at Hullabaloo, I was tasked with saving this project which had been heading towards disaster. With less than two-months to deliver this project (most of the budgeted time had already been used up in the financial year), and a previous code-base that was tangled, I made the bold choice to re-build the entire application with Nuxt.

My aim with this project was to deliver a robust build that could be amended and scaled easily with modular design patterns, since it was likely that there would be more products.

I also made heavy use of Git version control, which proved to be a vital decision since the client had later wanted the application to support 4 different languages (Chinese; Thai; Korean; Vietnamese), as well as 5 separate English versions for 5 different countries (Australia; China; Thailand; Singapore; India).

Since there were so many different versions, deployment to the staging and live environments became time-consuming and error-prone. Instead, I wrote shell scripts to automate this process.

Most of the code has been written from scratch, with the exception of the SCSS which has been inherited from the original PHP- and JQuery-based build. This is not as modular as it could be (albeit significant changes were made to the styles), and may be a point of improvement if the project needs to expand again, but due to time-constraints I had to compromise this.
